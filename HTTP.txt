Request Headers


User-Agent: It is simply the User-Agent
Accept: Informs the server about types of data that can be sent back.
Cache Control: Directives for caching mechanisms in both requests/responses
Postman-Token: Scans your public workspaces, collections, environment, and documentation to find exposed authentication tokens.
Host: Specifies the domain name of the server and optionally the TCP port
Accept-Encoding: Informs the server about encodings (compression algorithms) supported.
Connection: Indicates if the network connection stays open after the current transactions finished




Response Headers


Content_Type: Indicates the type of media in the resource
Vary: Sets which header fields should be considered as varying if a file is requested from the cache.
Content-Encoding: Tells us about the type of file.
Date: When the message was originated.
Server: Contains information about the software used by the origin server to handle the request
Cache-Control:  Directives for caching mechanisms in both requests/responses
X-XSS-Protection: feature that stops a page from loading it detects XSS attacks
X_Frame-Options: Prevent the site from clickjacking attacks
X-Content_Type-Options: act as a marker that indicates the MIME-types headers 
Alt-Svc: allows server to inform a web browser about alternative ways to reach your website
Transfer-Encoding: Hop By hop, message between two nodes
